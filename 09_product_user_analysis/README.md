## Mobile App User Behavior Analysis

## Project Overview

This project analyzes user behavior in a mobile application to understand engagement and conversion patterns.

The goal is to identify what drives user purchases and improve product performance.

---

## Dataset

The dataset was synthetically generated and includes:

- user_id — unique user ID  
- country — user location  
- device — device type (iOS / Android)  
- session_length — time spent in app  
- events — number of actions  
- purchase — target (0/1)  

---

## Tools Used

- Python
- pandas
- numpy
- matplotlib
- seaborn

---

## Key Metrics

- Average session length  
- Conversion rate  
- Conversion by device  
- Conversion by country  

---

## Key Steps

### 1. Data Analysis

- Average session length
- Overall conversion rate
- Conversion by device and country

---

### 2. User Segmentation

Users were divided into:

- Light users  
- Medium users  
- Heavy users  

Based on session length.

---

### 3. Behavioral Analysis

- Conversion by user segment
- Relationship between session length and purchase

---

## Visualizations

- Histogram: Session length distribution
- Bar chart: Conversion by device
- Bar chart: Conversion by country
- Bar chart: Conversion by segment
- Boxplot: Session length vs purchase

---

## Key Insights

- Users with longer sessions are more likely to convert.
- Device and country impact conversion rates.
- Heavy users generate the most value.
